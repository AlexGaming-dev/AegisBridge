# Contributing

Thank you for contributing to AegisBridge.

## Contribution Types

- Bug Reports
- Feature Requests
- Discussions about architecture, UX, and security
- Pull requests with improvements

## Workflow

1. Open an issue first, or start a discussion for larger changes.
2. Fork the repository and work in a feature branch.
3. Keep commits small, traceable, and focused.
4. Run local checks before pushing:
   - npm run lint
   - npm run build:win
5. Open a pull request with a clear description, screenshots (if UI-related), and test notes.

## Pull Request Checklist

- [ ] Scope and objective are clearly described
- [ ] No sensitive data committed
- [ ] Lint succeeds
- [ ] Build succeeds
- [ ] Documentation updated (README/SECURITY when relevant)

## Communication

- General questions: GitHub Discussions
- Concrete issues: GitHub Issues
- Security topics: See SECURITY.md
