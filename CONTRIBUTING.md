# Contributing

Vielen Dank fur deinen Beitrag zu AegisBridge.

## Beitragsarten

- Bug Reports
- Feature Requests
- Diskussionen zu Architektur, UX und Sicherheit
- Pull Requests mit Verbesserungen

## Vorgehen

1. Erstelle zuerst ein Issue oder starte eine Discussion fur groessere Anderungen.
2. Forke das Repository und arbeite in einem Feature-Branch.
3. Halte Commits klein, nachvollziehbar und thematisch getrennt.
4. Fuehre vor dem Push lokale Checks aus:
   - npm run lint
   - npm run build:win
5. Eroffne einen Pull Request mit klarer Beschreibung, Screenshots (falls UI) und Testnotizen.

## Pull Request Checklist

- [ ] Ziel und Umfang klar beschrieben
- [ ] Keine sensiblen Daten eingecheckt
- [ ] Lint erfolgreich
- [ ] Build erfolgreich
- [ ] Dokumentation aktualisiert (README/SECURITY falls relevant)

## Kommunikation

- Allgemeine Fragen: GitHub Discussions
- Konkrete Probleme: GitHub Issues
- Sicherheitsrelevante Themen: Siehe SECURITY.md
